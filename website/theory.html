From hand-crafted rules to scalable learning

Feature engineering and the Bitter Lesson

Before deep learning, computer-vision systems depended heavily on features designed by people. CNNs changed the division of labour: the model learned useful visual features directly from examples.

BEFORE DEEP CNNs

Hand-crafted features

Image→SIFT / HOG→Classifier

Researchers specified which edges, corners, gradients and textures the system should measure.

DEEP CNNs

Learned representations

Pixels→Convolutions→Features

Training discovered increasingly useful representations, from local patterns to class-relevant structures.

THE BITTER LESSON

General methods that scale

Data+Compute+Learning

Rich Sutton's argument is that scalable learning and search methods tend, over time, to outperform systems built around extensive human knowledge.

All benchmarks eventually come to an end

Rapid improvement can make a benchmark less able to distinguish the frontier. The classic ImageNet Large Scale Visual Recognition Challenge ran from 2012 to 2017; attention then moved towards harder tasks, larger datasets and new model families.

CNNs did not come to an end. What ended was the period in which annual gains by CNNs on the ImageNet classification benchmark largely defined progress in computer vision. CNNs remain widely useful, while Transformers and hybrid architectures changed where the frontier sits.

ImageNet and transfer learning

Large-scale supervised pretraining supplied reusable visual features. This project adapted those features to a much smaller binary classification task.

Why scale mattered

More labelled categories and images allowed deep networks to learn increasingly useful representations rather than depending entirely on hand-engineered features.

What transfers

Early and middle layers often detect edges, textures and shapes that remain useful beyond the original ImageNet classes.

What changes

The final classifier is replaced. Later, a small part of the pretrained base may be unfrozen and adapted cautiously.

CNN foundations

Six ideas that turn pixels into a prediction

Each box begins with the essential idea. Open it when you want the mechanism, an example and its role in this project.

01

Convolution and kernels

A small learned kernel slides across the image and produces a feature map.

Pixels→Kernel→Feature map
Open deeper lesson

What is being learned?

The kernel contains weights. At every image location, it combines nearby pixel values into one response. Training changes those weights so that useful patterns produce strong responses.

Why reuse the same kernel?

Weight sharing lets the same pattern be recognised anywhere in the image. It also requires far fewer parameters than connecting every pixel independently.

In this project

The pretrained kernels already detected useful visual structure learned from ImageNet. Fine-tuning cautiously adjusted only the deepest part of that learned system.

02

Activation functions

ReLU introduces non-linearity, allowing the network to learn complex visual relationships.

ReLU(x) = max(0, x)
Open deeper lesson

Why is non-linearity necessary?

Without activation functions, many stacked layers would still behave like one linear transformation. ReLU keeps positive responses and sets negative responses to zero.

What does that achieve?

Different filters can become active for different visual patterns. Later layers can then combine these selective responses into more complex features.

Do not confuse it with sigmoid

ReLU is commonly used inside the CNN. Sigmoid was used at this project’s final output to produce one restricted-class probability.

03

Pooling and stride

Feature maps shrink while important responses and broader spatial structure are retained.

Large map→Downsample→Compact map
Open deeper lesson

Two ways to reduce size

Pooling summarises a small neighbourhood, while a strided convolution moves the kernel by more than one pixel. Both reduce width and height.

Why reduce the map?

It lowers computation and lets later units gather evidence from a larger part of the original image. Some precise location information is deliberately traded for compactness.

Global average pooling

This project’s classification head averaged every final feature map to one number, creating a compact feature vector before dropout and the dense output.

04

Receptive fields

Deeper units combine evidence from progressively larger regions of the original image.

Open deeper lesson

Local first, broader later

An early activation depends on only a small patch. After several convolution and downsampling stages, one deep activation can be influenced by much more of the image.

Why depth matters

A deeper layer can combine edges into textures, textures into shapes and shapes into more class-relevant structures.

Effective versus theoretical field

A unit may theoretically receive information from a large area, but in practice some pixels influence it more strongly than others. Grad-CAM helps inspect where class-relevant deep activations occur.

05

Dense or fully connected layers

Every incoming feature connects through a learned weight to every neuron in the layer.

Dense(1) + sigmoid → probability
Open deeper lesson

What reaches the dense layer?

Not raw pixels. The CNN base has already converted the image into high-level feature signals, and global average pooling has compressed them into a vector.

What does “fully connected” mean?

Each feature value contributes through its own learned weight. A bias is added, producing the value passed to the activation function.

In this project

There was one output neuron. It learned how strongly the pooled features counted for or against the restricted class.

06

Sigmoid and threshold

Sigmoid produces a score from 0 to 1; a decision threshold converts it into a class.

0.5
Open deeper lesson

Score is not certainty

The output is a model score interpreted as a restricted-class probability, but it should not automatically be treated as perfectly calibrated confidence.

The threshold creates the label

At a threshold of 0.5, scores at or above 0.5 become restricted predictions. Lowering the threshold usually catches more restricted dogs but may create more false positives.

Why ROC-AUC and PR-AUC differ

A single confusion matrix uses one threshold. ROC and precision–recall curves evaluate scores across many possible thresholds.

Training methods

Six ideas that explain how fine-tuning worked

Follow the arc from error measurement and backward gradients to careful updates, regularisation and stopping.

01

Backpropagation

Gradients carry information about the loss backwards through every trainable operation.

Loss←Head←Unfrozen layers
Open deeper lesson

Forward first, backward second

The image passes forward to produce a score and loss. Backpropagation then applies the chain rule from the output towards earlier trainable layers.

What is a gradient?

It indicates how a small change in a weight would change the loss. The optimiser uses that information to choose an update direction.

Frozen means no update

Gradients were used to update the new head and, during fine-tuning, the final 20 unfrozen base layers. The earlier frozen layers retained their pretrained weights.

02

Dropout

Random activations are temporarily suppressed during training to reduce co-dependence.

Project setting: Dropout 0.20Open deeper lesson

What 0.20 means

During each training step, 20% of the selected activations were randomly set to zero. A different pattern can be dropped on the next step.

Why can this help?

The classifier cannot always depend on exactly the same feature combination, which can improve generalisation.

Training versus evaluation

Dropout is active during training but disabled during validation and testing. The framework scales the activations so predictions remain comparable.

03

Batch normalization

Intermediate activations are normalised and then given a learned scale and shift.

normalise → scale → shift
Open deeper lesson

Two kinds of state

Batch normalization has trainable scale and shift parameters, plus moving mean and variance statistics used at inference.

Why transfer learning needs care

A small new dataset may produce unstable moving statistics. A layer can therefore be technically trainable while batch-normalization behaviour is deliberately kept in inference mode.

Why architectures use it

It can stabilise activation distributions and make optimisation easier, especially in deep networks.

04

Fine-tuning

The final 20 base layers were unfrozen so deeper features could adapt to the dog task.

Open deeper lesson

It does not learn “in reverse”

The error signal travels backwards, but every trainable weight is updated according to its gradient. The deepest layers were chosen because they encode more task-specific combinations than early edge detectors.

Why not unfreeze everything?

The project dataset was much smaller than ImageNet. Restricting adaptation reduced the risk of destroying useful pretrained representations or overfitting.

What changed in the result?

Accuracy, precision and F1 rose, while restricted-class recall fell slightly. Fine-tuning changed the error trade-off; it did not improve every metric.

05

Learning rate and Adam

A low rate controlled the size of weight updates in the newly unfrozen layers.

Adam · 0.00001
Open deeper lesson

Step size matters

A rate that is too large can overwrite useful ImageNet features. A rate that is too small may adapt too slowly within the available epochs.

What Adam adds

Adam maintains moving estimates of gradient direction and magnitude, adapting effective updates for different parameters.

Why 1e-5?

Fine-tuning usually calls for more cautious updates than training a new classification head. Here, 1e-5 means 0.00001.

06

Validation and early stopping

Validation performance determined when to stop and which checkpoint to retain.

Train→Validate→Keep best
Open deeper lesson

Training data updates the model

The optimiser learns from training batches. Validation data checks performance between epochs but must not be used to make gradient updates.

What patience 4 means

Training could continue through four epochs without the monitored improvement before stopping, allowing for normal fluctuation.

Best is not necessarily last

The retained checkpoint came from the strongest monitored validation point. The held-out test set was then reserved for final evaluation.

Before and beyond CNNs

Before deep CNNs, systems often used hand-designed descriptors. Vision Transformers later represented images as patch tokens and learned relationships using self-attention.
