ImageNet and transfer learning

Large-scale supervised pretraining supplied reusable visual features. This project adapted those features to a much smaller binary classification task.

→
REUSEFrozen CNN basePreserve pretrained weights
→

→
REFINEFine-tuningAdjust the deepest features

Why scale mattered

More labelled categories and images allowed deep networks to learn increasingly useful representations rather than depending entirely on hand-engineered features.

What transfers

Early and middle layers often detect edges, textures and shapes that remain useful beyond the original ImageNet classes.

What changes

The final classifier is replaced. Later, a small part of the pretrained base may be unfrozen and adapted cautiously.

CNN foundations

A CNN learns local detectors, builds feature maps and combines them into progressively larger visual concepts.

01

Convolution and kernels

A small learned kernel slides across the image. Reusing the same weights at every location makes pattern detection efficient.

Pixels→Kernel→Feature map
02

Activation functions

Non-linear functions such as ReLU allow stacked layers to model more than one large linear transformation.

ReLU(x) = max(0, x)
03

Pooling and stride

Spatial dimensions shrink while important responses are retained. Deeper units therefore respond to larger effective regions.

Large map→Downsample→Compact map
04

Receptive fields

Early units see small local regions. Stacking layers lets deeper units combine evidence across more of the image.

05

Dense or fully connected layers

Every input feature connects to every neuron. Here, one sigmoid output combined pooled CNN features into a restricted-class probability.

Dense(1) + sigmoid → probability
06

Sigmoid and threshold

Sigmoid maps the output to 0–1. A threshold turns that score into a class label; changing it changes the confusion matrix.

Training and regularisation

Training changes weights to reduce loss. Regularisation and validation help prevent those changes from fitting the training images too narrowly.

01

Backpropagation

The loss measures prediction error. Gradients carry information backwards through trainable operations so the optimiser can update their weights.

Loss←Head←Unfrozen layers
02

Dropout

During training, a random proportion of activations is temporarily set to zero. This discourages the head from relying too heavily on one feature pathway.

Project setting: Dropout 0.20
03

Batch normalization

It normalises intermediate activations and learns a scale and shift. It can stabilise optimisation, but its moving statistics require care during transfer learning.

normalise → scale → shift
04

Fine-tuning

Unfreezing only the last 20 base layers allowed high-level features to adapt while earlier visual representations remained protected.

05

Learning rate

A low rate of 1e-5 made small weight updates, reducing the risk of rapidly overwriting useful pretrained features.

Adam · 0.00001
06

Early stopping

Validation performance was monitored and training stopped when improvement failed to continue. The best checkpoint was retained.

Train→Validate→Keep best

Before and beyond CNNs

Before deep CNNs, systems often used hand-designed descriptors. Vision Transformers later represented images as patch tokens and learned relationships using self-attention.

BEFORE DEEP CNNs

Engineered features

Image→SIFT/HOG→Classifier
CNN

Learned local hierarchy

Pixels→Filters→Features
VISION TRANSFORMER

Patch relationships

Patches→Attention→Tokens
The transition is not a simple replacement: CNNs remain powerful and efficient. Transformers offered a different route to scale and global relationships.
