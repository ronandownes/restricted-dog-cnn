Plain-language evaluation guide

What do precision and recall actually mean?

Begin with the model’s four possible outcomes. Every headline metric is simply a different way of combining those four counts.

Show me the four outcomesReturn to the benchmark
FIRST IMPORTANT POINT

“Positive” does not mean “good”.

In this project, positive means the model predicted “restricted”. Negative means it predicted “unrestricted”. True and false tell us whether that prediction matched the dataset label.

01 · The foundation

One image. Four possible outcomes.

The examples below use the fine-tuned InceptionResNetV2 test results. The restricted class is treated as positive.

02 · The headline metrics

Each metric asks a different question.

There is no single universally “best” metric. The right one depends on which error matters and whether the classes are balanced.

Accuracy97.96%

How often was the model correct overall?

Count every correct prediction—restricted and unrestricted—and divide by all images.

(TP + TN) ÷ all images
(183 + 824) ÷ 1,028 = 97.96%
Useful as a broad summary, but it can look impressive when one class is much larger than the other.
Precision92.89%

When the model said “restricted”, how often was it right?

Precision focuses on the reliability of positive predictions. False positives lower precision.

TP ÷ (TP + FP)
183 ÷ (183 + 14) = 92.89%
High precision means relatively few unrestricted images were incorrectly labelled restricted.
Recall96.32%

Of all restricted images, how many did the model find?

Recall focuses on coverage of the positive class. False negatives lower recall.

TP ÷ (TP + FN)
183 ÷ (183 + 7) = 96.32%
High recall means relatively few restricted-class images were missed.
F1 score94.57%

How strong is the balance between precision and recall?

F1 is the harmonic mean of precision and recall. It falls if either one is weak.

2 × (precision × recall) ÷ (precision + recall)
Balance of 92.89% precision and 96.32% recall = 94.57%
Useful when both false positives and false negatives matter.
PR-AUCAcross thresholds

How well does the model trade precision against recall?

A classifier produces scores, not just one final label. Changing the decision threshold changes precision and recall. PR-AUC summarises that relationship across many thresholds.

Area under the precision–recall curve Especially informative when the positive class is less common. It is not calculated from just one confusion matrix.
MCC0.9334

How well do predictions agree with labels across all four outcomes?

Matthews correlation coefficient uses TP, TN, FP and FN together. It ranges from −1 to +1.

−1
opposite0
no better than chance+1

The project’s 0.9334 indicates very strong agreement on this held-out test set—not legal or real-world validity.

03 · Reading the trade-off

Fine-tuning improved the balance, but not every metric.

The fine-tuned model became more precise while becoming slightly less sensitive to the restricted class.

↓ 14

Fewer false positives

False positives fell from 28 to 14. Precision therefore rose substantially.

↑ 2

More false negatives

False negatives rose from 5 to 7. Recall therefore fell slightly.

↑

Better overall balance

Accuracy, precision, F1 and MCC improved, despite the small recall reduction.

If false accusations matter mostWatch precision and false positives.
If missed restricted cases matter mostWatch recall and false negatives.
If both errors matterUse F1, MCC and the confusion matrix together.
The key lessonNever report a single metric without asking which mistakes it hides.
